## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (e.g. gap-1 between icon and label in nav) |
| sm | 8px | Compact element spacing (e.g. gap-2 between list card internals; py-2 list card vertical padding) |
| md | 16px | Default element spacing (px-4 page container horizontal padding) |
| lg | 24px | Section padding (mb-6 between header and list) |
| xl | 32px | Layout gaps (pt-8 page top padding) |
| 2xl | 48px | Empty state vertical padding (py-16 = 64px — see exception below) |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Empty state uses `py-16` (64px vertical) — matches existing placeholder in `src/app/(app)/page.tsx`.

Platform Constraints (not spacing tokens — not subject to 8-point scale):
- Bottom nav height: 56px (`h-14`) — matches existing `(app)/layout.tsx` safe-area pattern; 56px is a multiple of 4 and matches the established layout height. Value is platform-matched, not a spacing scale token.
- Touch targets on bottom nav items: minimum 44px height — iOS tap target standard (platform constraint, not a design token).

Source: `src/app/(app)/page.tsx` placeholder + `src/app/(app)/layout.tsx` safe-area pattern.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px (`text-base`) | 400 (regular) | 1.5 |
| Label / secondary | 14px (`text-sm`) | 400 (regular) | 1.4 |
| Heading | 20px (`text-xl`) | 600 (semibold) | 1.2 |
| Badge / timestamp | 12px (`text-xs`) | 400 (regular) | 1.3 |

Notes:
- All page headings use `text-xl font-semibold text-white` — matches `src/app/(app)/page.tsx` (`h1 className="text-xl font-semibold text-white"`).
- Secondary/muted text uses `text-white/65` — established pattern from output page and placeholder.
- Asset type/subtype subtitle on cards uses `text-sm text-white/65` — matches output page subtitle pattern.
- Badge text uses `text-xs` at weight 400 (regular) — the 12px size and muted/navy background provide sufficient visual distinction without a third weight.
- 4 sizes declared (12, 14, 16, 20); 2 weights declared (400 regular + 600 semibold).

## Interaction Contract

### Bottom Navigation

- Fixed to bottom of viewport on all `(app)` route pages.
- Height: 56px (`h-14`) plus `pb-[env(safe-area-inset-bottom)]`.
- Background: `bg-[#14532D]` (secondary/card color) with top border `border-t border-[#1E3A5F]`.
- Two items: "Assets" (List icon + label) and "New Asset" (Plus icon + label).
- Active state: current tab uses `text-white font-semibold`; inactive uses `text-white/65`.
- Active tab indicator: `bg-[#1E3A5F]` rounded pill behind the active item OR a 2px `border-t-2 border-white` top accent on the active item.
- "Assets" tab is active when pathname is `/`; "New Asset" tab is active when pathname starts with `/assets/new`.
- On all other routes (`/assets/[id]/*`), neither tab shows as active — both render at `text-white/65`.

### Branch Selector (first visit)

- Shown when `localStorage('lastUsedBranch')` is null/missing.
- Display: full-screen overlay above the list (not a modal — replaces main content area).
- Heading: "Select your branch" (`text-xl font-semibold text-white`).
- Body: "Your branch is saved — you can change it any time." (`text-sm text-white/65`).
- Reuse `BranchSelector` component — 10 branch buttons, existing styling (`border-2 border-[#1E3A5F] bg-[#14532D]`, selected `border-white bg-white/10`).
- On select: write to `localStorage('lastUsedBranch')`, dismiss screen, render asset list.

### Branch Header (return visit)

- Rendered at top of list page below page heading.
- Shows active branch label as a chip/button with `ChevronDown` icon.
- Tap: opens an inline dropdown or sheet to change branch.
- On change: update `localStorage('lastUsedBranch')`, re-fetch list filtered to new branch.

### Asset List Cards

- Each card is a full-width tappable link (`<Link>`) wrapping an `AssetCard`.
- Card padding: `px-4 py-2`.
- Card background: `bg-[#14532D]` (card token).
- Card border: `rounded-lg border border-[#1E3A5F]`.
- Card layout (vertical stack):
  - Row 1: Asset type label (`text-sm text-white/65 uppercase tracking-wide`) + `AssetStatusBadge` (right-aligned)
  - Row 2: Make / model / year (`text-base text-white font-medium`) — or "No data yet" (`text-white/65 italic`) when `fields` is empty/null
  - Row 3: Relative timestamp (`text-xs text-white/65`) — e.g. "2h ago", "yesterday", "3 days ago"
- Cards separated by `gap-3` in the list container.
- Tap target: entire card is a link (no nested buttons).

### Resume Routing (from card tap)

- Draft asset → `/assets/[id]/review`
- Confirmed asset → `/assets/[id]/output`
- Logic lives in `AssetCard` link href, derived from `asset.status` column.

### "Book In New Asset" (output page — confirm + navigate)

- Tapping "Book In New Asset" on the output page must first flip `assets.status` to `'confirmed'` via a Server Action, then navigate to `/assets/new`.
- Until this Server Action is added, the existing `<Link href="/assets/new">` is a placeholder.
- The confirmation flips status atomically — no intermediate loading state required (fast DB write).

### Empty State (branch has no assets)

- Shown when Supabase query returns 0 records for the selected branch.
- Layout: centered vertically in remaining viewport space, `py-16`.
- Heading: "No assets yet" (`text-base font-medium text-white`).
- Body: "Tap New Asset to start booking in an asset." (`text-sm text-white/65 mt-2`).
- Source: existing placeholder in `src/app/(app)/page.tsx` (reuse verbatim).
