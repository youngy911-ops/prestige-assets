FIXED:
```typescript
    shape[field.key] = field.inputType === 'number'
      ? z.string().regex(/^\d*\.?\d*$/, 'Must be a number').or(z.literal(''))
      : z.string()
```

This is a one-character change: insert `\.?` between the two `\d*` atoms. The anchors (`^` and `$`) and `.or(z.literal(''))` union are unchanged.

Rationale: `/^\d*\.?\d*$/` allows an optional single decimal point with digits on either side. The `.or(z.literal(''))` already handles the empty-string case. Strings like `'.'` alone will pass this regex — this is acceptable (they are not valid numbers in practice and the form's UX prevents entering a lone period from persisting).

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-24-01 | Tampering | `build-form-schema.ts` regex | accept | New regex `/^\d*\.?\d*$/` is still anchored at both ends; it cannot be bypassed by injection. Accepts only digits and one optional period — letters, symbols, and control characters are still rejected. The `.or(z.literal(''))` union only passes the empty string, not arbitrary input. |
| T-24-02 | Tampering | `agriculture.ts` aiHint | accept | aiHint is a static string baked into the TypeScript bundle at build time; it is never user-controlled input. No attack surface introduced. |
| T-24-03 | Denial of Service | Regex backtracking | accept | `/^\d*\.?\d*$/` has no nested quantifiers or catastrophic backtracking risk. Both `\d*` atoms are non-overlapping with `\.?`; linear scan. No risk of ReDoS. |
| T-24-04 | Information Disclosure | `inputMode="decimal"` | accept | Changing the mobile keyboard type from numeric to decimal exposes no new information; it only changes which keyboard variant the OS displays.
