## Interaction Contracts

### Page Load — Description Auto-Generation

1. Page renders with fields block immediately (synchronous — data from `assets.fields` JSONB).
2. Description block renders in loading state simultaneously (spinner + "Generating description…" label).
3. `OutputPanel` fires POST `/api/describe` on mount (useEffect).
4. On success: description populates editable textarea. Textarea is immediately editable.
5. On failure: auto-retry once. If retry fails: show error message + empty editable textarea.
6. Fields block is always visible and copyable — description failure does not block the fields block.

### Loading State (description block)

- Spinner: use `lucide-react` `Loader2` icon with `animate-spin` class — consistent with Phase 3 extraction loading pattern.
- Label: "Generating description…" in text-sm text-white/65.
- Description block card renders at its full height with a skeleton placeholder so the page does not jump when content loads.

### Copy-to-Clipboard Confirmation

Treatment: **Inline label change** on the copy button.
- Default label: "Copy Fields" / "Copy Description" (with `Copy` lucide icon).
- On click: label changes to "Copied!" (with `Check` lucide icon) for 2000ms, then reverts.
- No toast. Inline change is sufficient for a single-action confirmation and avoids toast z-index conflicts.
- Button remains clickable during the "Copied!" state (re-copy allowed immediately).

### Regenerate Button (description block)

- Visible below the description textarea at all times (not hidden post-generation).
- Label: "Regenerate" with `RefreshCw` lucide icon.
- If textarea has been manually edited: show browser-native `confirm()` dialog — "Your edits will be lost. Regenerate description?" — before firing the API call.
- If textarea has NOT been edited: fires immediately without confirmation.
- During regeneration: textarea becomes read-only, Regenerate button shows spinner + "Regenerating…" label.

### Empty / Null Fields in Structured Fields Block

Treatment: **Show blank value** — include every field from Schema Registry `sfOrder` with an empty value line, never omit the field label.

Rationale: Salesforce operators paste the entire block. Omitting fields creates a mismatched template. Blank lines signal "not provided" without ambiguity.

Format per field line: `{Salesforce Label}: {value or blank}`

### Navigation

- **Back link**: chevron-left icon + "Review" label, top-left, text-white/65, hover text-white. Routes to `/assets/{id}/review`. Consistent with review page back link pattern.
- **New Asset button**: bottom of page, after both output blocks. Uses Button component with secondary variant. Routes to `/assets/new`. Label: "Book In New Asset".

### Description Persistence

Generated description is persisted to `assets.description` (new DB column) on successful generation. On revisit, if `assets.description` is non-null, the saved description is loaded directly — no GPT-4o call fired. Staff can still click "Regenerate" to get a fresh generation.

## Layout Contract

Page route: `/assets/[id]/output`
Container: `max-w-2xl mx-auto px-4 pt-8 pb-[calc(env(safe-area-inset-bottom)+80px)]` — identical to review page container.

Block order (top to bottom):
1. Header row — back link + page title "Output" + asset type/subtype meta
2. Salesforce Fields block (Card) — always renders immediately
3. Description block (Card) — renders loading state on mount, then populates
4. New Asset button — full-width, below both blocks, mt-8

Each block is a Card with internal padding of 16px (md). Copy buttons sit at the bottom-right of each card. Section header labels sit at the top-left of each card in text-sm font-semibold text-white/65.

Gap between blocks: 24px (lg spacing token).
