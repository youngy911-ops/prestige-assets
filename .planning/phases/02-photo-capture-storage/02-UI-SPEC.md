## Interaction Contract

### File Picker (PHOTO-01)
- Trigger: Button labelled "Add Photos" with `Camera` lucide icon (left of label)
- Input element: `<input type="file" accept="image/*" multiple capture="environment">` — hidden, triggered via button click
- On mobile: opens camera roll / camera; on desktop: opens OS file picker
- Disabled state: button is `opacity-40` while any upload is in-progress (matches Phase 1 pattern)

### Upload Flow (PHOTO-02)
- Client-side processing before upload: resize to max 2MP via `browser-image-compression`, EXIF orientation correction via `exifr`
- Progress: each thumbnail shows a semi-transparent overlay with a `Loader2` spin icon while uploading (matches Phase 1 loading pattern)
- On upload complete: overlay removed, thumbnail shows full image
- On upload error: overlay replaced with `#F87171` error tint + `AlertCircle` icon; error text shown below grid

### Drag-to-Reorder (PHOTO-03)
- Library: `@dnd-kit/sortable` + `@dnd-kit/core`
- Drag handle: `GripVertical` lucide icon, visible on each thumbnail (16×16px, `text-white/65`, hover `text-white`)
- Active drag: dragged thumbnail renders at 90% opacity; drop target slot shows accent-colored dashed border
- Cover photo: always the first thumbnail (index 0) after any reorder; cover badge updates instantly on drop
- Persistence: reorder triggers a Server Action to update `position` column in `asset_photos` table
- Disabled during: any active upload in the grid

### Empty State (no photos yet)
- Layout: centred column, `Camera` icon (48×48px, `text-white/40`), heading, subheading, "Add Photos" button
- Heading: "No photos yet" (28px/semibold — Display role)
- Subheading: "Add photos from your camera roll or file system." (14px/regular, white/65)
- Button: full-width, accent background, "Add Photos"

### Thumbnail Grid (photos present)
- Layout: 3-column grid on mobile, 4-column on desktop (≥768px)
- Thumbnail size: 80×80px minimum, aspect-ratio 1/1, `object-fit: cover`
- Cover badge: absolute top-left, `Badge` variant `secondary`, text "Cover", accent border
- Remove button: absolute top-right `X` icon button, `text-white/65` idle, `text-destructive` on hover, 32×32px touch area

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | "Add Photos" (with Camera icon) |
| Section heading (photos present) | "Photos" |
| Section subheading (photos present) | "{N} photo{s} — drag to reorder" |
| Empty state heading | "No photos yet" |
| Empty state body | "Add photos from your camera roll or file system." |
| Upload error (generic) | "Upload failed. Check your connection and try again." |
| Upload error (file type) | "Only image files are supported." |
| Upload error (size) | "That file is too large to upload." |
| Reorder persistence error | "Failed to save photo order. Reload the page and try again." |
| Remove confirmation | No confirmation dialog — remove is immediate. Undo is not in scope for Phase 2. |
| Cover badge label | "Cover" |
| Drag handle aria-label | "Drag to reorder" |
| Remove button aria-label | "Remove photo" |

Note on destructive actions: photo removal is immediate with no confirmation modal. The interaction is low-stakes (re-upload is possible) and a confirmation dialog would disrupt the rapid multi-photo upload flow on mobile. This is a Phase 2 decision; a soft undo toast could be added in a later phase.

Source: REQUIREMENTS.md PHOTO-01, PHOTO-02, PHOTO-03 + Phase 1 copy style (direct, no marketing language)
