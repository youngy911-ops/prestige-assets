## Interaction Contract

This section is specific to Phase 11 and documents the before/after for each interaction being fixed. There are no new interactive elements — only corrected runtime behaviour for existing ones.

### 1. Structured Input Fields (VIN, odometer, hourmeter, unladen weight, length)

| Property | Before (bug) | After (fixed) |
|----------|-------------|---------------|
| Display on load | Empty (no `defaultValue` wired) | Shows previously entered value |
| Placeholder visibility | Always shown | Hidden when value is present |
| Visual change | None | None — same styling, populated DOM value |

Implementation contract: each `<Input>` receives `defaultValue={structuredValuesRef.current[field.key] ?? ''}`. No style changes. Source: CONTEXT.md / RESEARCH.md Pattern 3.

### 2. Suspension Type Select

| Property | Before (bug) | After (fixed) |
|----------|-------------|---------------|
| Display on load | Blank placeholder always shown | Shows previously selected option (e.g. "Airbag") |
| Primary approach | n/a | Uncontrolled: `defaultValue={structuredValuesRef.current[field.key] ?? undefined}` |
| Fallback approach | n/a | Controlled: `value={selectValues[field.key] \|\| undefined}` + `useState` if `defaultValue` renders blank |

Placeholder copy when no value saved: "Select suspension type" (from `FIELD_PLACEHOLDERS.suspension` — unchanged). Source: CONTEXT.md / RESEARCH.md Pattern 4.

### 3. "Other notes" Textarea

| Property | Before (bug) | After (fixed) |
|----------|-------------|---------------|
| `defaultValue` content | Full raw `inspection_notes` blob (e.g. `"vin: ABC123\nNotes: runs well"`) | Freeform notes only (e.g. `"runs well"`) |
| Placeholder | "VIN, rego, dimensions…" | Unchanged |
| Visual change | None | None — same styling, corrected DOM value |

Source: CONTEXT.md (textarea freeform notes decision).

### 4. Unmount Flush (Fast Navigation)

| Property | Behaviour |
|----------|-----------|
| Trigger | Component unmount (Next.js router navigation, back button) |
| Action | Cancel debounce timer (`clearTimeout(debounceRef.current)`) then call `persistNotes()` synchronously |
| Visual feedback | None — silent autosave, consistent with existing UX |
| User-visible change | In-flight edits are no longer silently lost on fast navigation |

Source: CONTEXT.md (unmount flush decision). No UI change — purely invisible correctness fix.

## Phase-Specific Notes

### What changes visually

On load, the `InspectionNotesSection` card will show populated field values instead of empty inputs and a blank Select trigger. This is the entire user-visible change. All colours, spacing, typography, borders, and layout are identical to the current state.

### What does NOT change

- Card structure and layout
- Label styling (`text-xs text-white/65`)
- Input styling (`h-9 text-sm bg-white/5 border-white/15 text-white`)
- Select trigger styling
- Textarea styling (`w-full min-h-[80px] rounded-md border border-white/15 bg-white/5`)
- Focus ring colour (`focus:ring-[oklch(0.29_0.07_248)]`)
- Placeholder text on any field
- Card header title "Inspection Notes"

### Ref seeding contract

`structuredValuesRef.current` is seeded directly in the `useRef()` call — not in a `useEffect` — to avoid a blank-then-populated flash on mount. Same for `notesRef.current`. This is an invisible implementation detail with no visual consequence.
