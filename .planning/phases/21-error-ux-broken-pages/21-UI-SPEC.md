## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap, inline element separation |
| sm | 8px | Button stack gap, detail toggle margin |
| md | 16px | Default section padding, card inner padding |
| lg | 24px | Vertical rhythm between error heading, body, and action row |
| xl | 32px | Top padding for error page content block (auth: pt-24 = 96px; app: pt-16 = 64px) |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | App error boundary top padding (pt-16 = 64px, per existing pattern) |

Exceptions:
- Auth error page top padding: 96px (pt-24) — matches existing login page layout, source: RESEARCH.md
- App error page top padding: 64px (pt-16) — matches existing app shell layout, source: RESEARCH.md
- Touch targets for recovery action buttons: minimum 44px height via `size="lg"` on Button component

## Color

All color values below are the semantic Tailwind tokens as established by Phase 20. Raw hex is PROHIBITED in `(app)/error.tsx` and `(auth)/error.tsx`. `global-error.tsx` uses inline styles only (CSS not loaded at that boundary).

| Role | Semantic Token | OKLCH value | Usage |
|------|---------------|-------------|-------|
| Dominant (60%) | `bg-background` | oklch(0.16 0.04 148) | Page background for both error pages |
| Secondary (30%) | `bg-card` | oklch(0.20 0.05 148) | ErrorDisplay container / card panel |
| Accent (10%) | `bg-primary` / `text-primary` | oklch(0.52 0.17 148) | Primary recovery buttons (default variant) only |
| Destructive | `text-destructive` | oklch(0.74 0.15 20) | Error icon, error heading text colour |

Accent reserved for: primary recovery action buttons ("Sign in again", "Go to Assets") — and nothing else on these pages.

- `text-muted-foreground` — subdued body copy and detail toggle label (55% white)
- `border-border` — card/panel border (8% white)

**global-error.tsx inline style hex values (inline styles only, Tailwind unavailable):**
- Background: `#0f1f0f` (brand deep dark green, already correct from Phase 20)
- Text primary: `#ffffff`
- Text muted: `rgba(255,255,255,0.6)`
- Do NOT introduce new hex values — use only those already established

## Layout Patterns

### ErrorDisplay shared component
- Centered column layout, `max-w-[480px]` for app errors, `max-w-[360px]` for auth errors (pass via `className` prop)
- Vertical stack: icon → heading → body message → action row → detail toggle
- Action row: horizontal flex, gap-8 (8px), wraps on small screens
- Detail toggle: native `<details><summary>` element — zero JS, correct semantics

### global-error.tsx layout
- Full-page centered, inline styles only
- Stack: heading → body message → home link
- No ErrorDisplay import — this file cannot import components (CSS not loaded, must be self-contained)

Source: RESEARCH.md architecture section

## Interaction Contract

| Interaction | Behaviour |
|-------------|-----------|
| "Sign in again" button | `<Link href="/login">` wrapped in `<Button asChild variant="default" size="lg">` |
| "Go to Assets" button | `<Link href="/assets">` wrapped in `<Button asChild variant="default" size="lg">` |
| "Try Again" button | `<Button variant="outline" size="lg" onClick={reset}>` where `reset` is the Next.js error boundary prop |
| "Show details" toggle | Native `<details><summary>` — no JS state. Collapsed by default. Shows `error.message` and `error.digest` (if present) in `<pre className="font-mono text-xs text-muted-foreground">` |
| Edit-type route | Deleted — no interaction. Zero inbound links, no redirect needed. |

Touch targets: All buttons use `size="lg"` which renders at minimum 44px height.
