### Select visual contract

The Select trigger must visually match the existing Input styling:

```
className="h-9 text-sm bg-white/5 border-white/15 text-white"
```

- Height: 36px (h-9) — matches Input height
- Background: `bg-white/5`
- Border: `border-white/15`
- Text: `text-white`
- SelectValue placeholder text: `text-white/30`
- Focus ring: `focus:ring-[oklch(0.29_0.07_248)]`
- Dropdown items: inherit card background (`--card`) with `text-white` foreground

Source: InspectionNotesSection.tsx Input className; RESEARCH.md Pattern 2 example.

### Visual distinction: select vs text input

Suspension Type (select) is visually distinguished from text inputs by:
1. The SelectTrigger renders a chevron-down icon (built into Base UI Select) on the right edge
2. No other visual distinction is needed — the chevron is sufficient affordance for a dropdown

No additional indicator, badge, or colour change is needed to mark select fields differently from text inputs.

Source: Claude's Discretion (CONTEXT.md) — "how to visually distinguish pre-extraction fields from the freeform notes textarea"; chevron inherent to Base UI Select component.

## Interaction Contract

### Autosave behaviour (existing — unchanged)

All structured field changes trigger a 500ms debounced `saveInspectionNotes` call. No save button. No save confirmation toast. Behaviour is invisible to staff — same as the existing pattern.

### Select interaction (Suspension Type)

- Default state: SelectTrigger renders placeholder "Select suspension type" at 30% white opacity
- Open state: SelectContent drops below trigger; items Spring / Airbag / 6 Rod / Other listed vertically
- Selected state: SelectTrigger shows selected value at full `text-white` opacity
- Value change fires `handleStructuredChange(field.key, value)` immediately (no debounce on the select itself — autosave debounce handles persistence)
- Uncontrolled: no `value` prop; no `defaultValue` (re-hydration deferred to v1.2); Select starts empty on every mount

### No re-hydration (Phase 9 scope)

Pre-extraction field values are NOT restored when staff navigate away and return. Inputs mount empty. This is explicitly deferred to PREFILL-06 in v1.2. No UI indicator needed.

Source: CONTEXT.md Deferred; RESEARCH.md Anti-Patterns.
