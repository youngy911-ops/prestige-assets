The Ask

Sandbox access + approval to build Salesforce API integration.

Today: copy-paste ready output. Goal: direct write into Salesforce — no copy-paste step.

Specifically requesting:

  1. Salesforce sandbox access for development and testing
  2. External Client App in sandbox — OAuth 2.0 Web Server Flow (Consumer Key + Secret)
  3. Defined pilot scope — one site, 2 weeks, small user group
  4. Review meeting at end of pilot to decide on production rollout
Risk & Governance

The hard questions, answered.

Data & privacy

  • Photos → Supabase, Sydney (au-southeast-2)
  • AI call → OpenAI. Not used for training. AU data residency available.
  • All Salesforce writes carry the user's identity — native audit trail
  • Zero Data Retention (ZDR) available via enterprise agreement

If something goes wrong

  • Bad photos → flags missing fields, doesn't invent data
  • Wrong AI extraction → human review step catches it before any write
  • OpenAI outage → clear error, fall back to manual entry
  • Vendor risk → extraction layer is modular, swap providers without rewrite

The Bigger Picture

The book-in app is just the start.
The same foundation unlocks AI automation across Salesforce.

🏷️
Auction creation
AI reads a sale brief and creates the full auction structure in Salesforce — dates, lots, assignments. What takes an admin an hour, done in seconds.
📬
Consignor updates
AI monitors asset status and automatically emails consignors when their asset is listed, sold, or settled — no manual chasing.
📄
Settlement automation
Post-sale, AI generates settlement statements from Salesforce lot data, attaches them to records, and triggers the finance workflow.
🎙️
Voice-to-Salesforce
Field staff record a voice note walking around an asset. AI transcribes it and populates the right Salesforce fields — no typing on site at all.

The External Client App approved today is the authentication layer for all of this. One approval — built once — reused across every future workflow.
