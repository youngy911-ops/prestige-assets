## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Gap between confidence icon and field value, inline element spacing |
| md | 16px | Default element spacing, card inner padding |
| lg | 24px | Section padding, gap between inspection notes block and CTA |
| xl | 32px | Layout gaps between major page sections |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level bottom padding |

Exceptions:
- Touch targets for CTA buttons: `h-11` (44px) — matches existing photos page pattern
- Page outer padding: `px-4` (16px) horizontal, `pt-8` (32px) top — inherits established layout
- Page max-width: `max-w-[480px] mx-auto` — matches existing photos page
- `12px`: compact field row density within card; used as `py-12px` on result rows and as gap between inspection fields — smaller than `md` (16px) to keep the list scannable without excess whitespace
- `80px`: safe-area sticky footer clearance; used in `pb-[calc(env(safe-area-inset-bottom)+80px)]` to prevent the sticky CTA from obscuring the last result row on iOS

Source: `src/app/(app)/assets/[id]/photos/page.tsx` established pattern

## Typography

Two weights only: 400 (regular) and 600 (semibold).

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (text-sm) | 400 (regular) | 1.5 |
| Label | 12px (text-xs) | 400 (regular) | 1.4 |
| Heading | 20px (text-xl) | 600 (semibold) | 1.2 |
| Display | 28px (text-2xl) | 600 (semibold) | 1.2 |

Notes:
- Page heading uses `text-xl font-semibold` — matches existing `/photos` page `h1`
- Field labels in the extraction result list use `text-sm` weight 400
- Field values in the extraction result list use `text-sm font-semibold` when extracted, weight 400 muted when not found
- Subheading / section label inside inspection notes block uses `text-sm font-semibold` (600) — not `font-medium`; the two-weight budget is 400 + 600 only
- Display size (28px) reserved for prominent extraction-complete summary if needed; default is heading (20px)

Source: `src/app/(app)/assets/[id]/photos/page.tsx` existing `text-xl font-semibold` / `text-sm` pattern

## Color

All values use CSS variable tokens (oklch). Do NOT use raw hex. Tailwind v4 with oklch color space — use `var(--*)` or Tailwind class equivalents.

| Role | CSS Token | oklch Value | Usage |
|------|-----------|-------------|-------|
| Dominant (60%) | `var(--background)` | `oklch(0.38 0.1 148)` | Page background, full-screen surfaces |
| Secondary (30%) | `var(--card)` | `oklch(0.34 0.1 148)` | Inspection notes card, extraction result panel, field rows |
| Accent (10%) | `var(--primary)` | `oklch(0.29 0.07 248)` | See reserved list below |
| Destructive | `var(--destructive)` | `oklch(0.74 0.15 20)` | "Try Again" error state, API failure banner |

Accent (`var(--primary)` — navy) reserved for:
1. "Run AI Extraction" primary CTA button background
2. "Proceed to Review" primary CTA button background
3. Border/ring on extraction stale banner (already established in Phase 2)
4. Focus ring on inspection notes textarea and structured input fields

Confidence indicator colors (semantic, not accent):
- High confidence: `text-green-400` / `oklch(0.79 0.17 150)` — green checkmark icon + value text
- Medium confidence: `text-amber-400` / `oklch(0.83 0.16 84)` — amber dot icon + value text
- Low / not found: `text-white/40` (`oklch(1 0 0 / 40%)`) — muted dash or warning icon

Muted text (secondary labels, helper copy): `text-white/65` (`oklch(1 0 0 / 65%)`)

Source: `src/app/globals.css` color token definitions + `src/app/(app)/assets/[id]/photos/page.tsx` existing usage pattern

## State Machine

The `/assets/[id]/extract` page renders exactly one of four states. No state overlap.

| State | Condition | UI |
|-------|-----------|-----|
| Trigger | `extraction_result IS NULL` | InspectionNotesSection + ExtractionTriggerState |
| Loading | Extraction POST in-flight | ExtractionLoadingState (replaces trigger CTA area) |
| Result | `extraction_result IS NOT NULL` | InspectionNotesSection (collapsed/summary) + ExtractionResultPanel |
| Failure | Route Handler returned error | ExtractionFailureState |

The `/photos` page gains InspectionNotesSection (notes + structured fields) above the existing "Run AI Extraction" CTA.

## Interaction Contracts

### Inspection Notes Auto-Save
- Debounce: 500ms after last keystroke
- Saves to `assets.inspection_notes` via Server Action + `revalidatePath`
- No visible save indicator (silent, consistent with photo ordering pattern)
- Structured priority fields each save independently on `onChange` with same debounce

### "Run AI Extraction" CTA
- Location: photos page (`/assets/[id]/photos`) — below photo grid and inspection notes
- Behaviour: POST to `/api/extract`; client transitions to loading state immediately
- Staff can navigate away during extraction — route handler completes in background
- On return to `/assets/[id]/extract`: if `extraction_result` exists, show result state directly

### Confidence Indicators
- Every Salesforce field for the asset subtype appears in the result list (extracted or not)
- Fields in `sfOrder` (ascending), flat scrollable list — no grouping, no accordion
- Each row: [field label] [value or "Not found"] [ConfidenceBadge]
- ConfidenceBadge: inline `<Badge>` with icon + text variant:
  - `high` → green, `CheckCircle2` icon, label "High"
  - `medium` → amber, `AlertCircle` icon, label "Medium"
  - `low` → muted white/40, `MinusCircle` icon, label "Low"
  - `not_found` → muted white/40, `MinusCircle` icon, label "Not found"

### "Proceed to Review" CTA
- Full-width, `h-11`, navy background — matches existing CTA button style on photos page
- Sticky to bottom of viewport on mobile when result list is long (position: sticky bottom-0)
- "Re-run Extraction" appears as a small text link (`text-sm text-white/65 hover:text-white`) below the sticky CTA

## Layout Spec

### Photos Page Addition (InspectionNotesSection)

Focal point: the "Run AI Extraction" primary CTA button — full-width navy, `h-11`, positioned immediately below the inspection notes card. It is the only high-contrast interactive element on this page surface and acts as the visual terminus of the form flow.

```
max-w-[480px] mx-auto px-4

[existing photo grid]
[gap: 24px]
[InspectionNotesSection card]
  heading: "Inspection Notes"  (text-sm font-semibold, mb-8px)
  [structured priority fields — up to 5, asset-type driven]
    each: Label (text-xs text-white/65) + Input (text-sm h-9)
    gap between fields: 12px
  [gap: 12px]
  Textarea "Other notes"
    Label (text-xs text-white/65) + Textarea (text-sm min-h-[80px])
[gap: 24px]
["Run AI Extraction" CTA — h-11 full-width navy button]
["Skip to Manual Entry" — text-sm text-white/65 text-center mt-8px]
```

### Extract Page — Result State

Focal point per state:
- Trigger state: the "Run AI Extraction" button (`h-11` full-width navy) — sole high-contrast element; draws the eye to the primary action.
- Result state: the "Proceed to Review" sticky CTA (`h-11` full-width navy, pinned to bottom) — remains visible as the user scrolls the field list and anchors the completion action.

```
max-w-[480px] mx-auto px-4 pt-8 pb-[calc(env(safe-area-inset-bottom)+80px)]

[header — same as photos page: back chevron + h1 "AI Extraction" + subtype line]
[gap: 24px]
[ExtractionResultPanel — scrollable field list]
  each row: flex justify-between items-center py-12px border-b border-white/10
    left: field label (text-sm text-white/65)
    right: value (text-sm font-semibold text-white) OR "Not found" (text-sm text-white/40)
    far right: ConfidenceBadge
[sticky bottom CTA area — bg-[var(--background)] pt-12px pb-safe]
  "Proceed to Review" full-width h-11 navy button
  "Re-run Extraction" text-sm text-white/65 text-center mt-8px
```
