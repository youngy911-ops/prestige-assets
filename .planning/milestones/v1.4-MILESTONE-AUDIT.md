### Integration Gaps (Non-Critical)

**Gap 1 — motor_grader / GRADER heading mismatch (DESCR-04)**

The `motor_grader` schema key sends `Subtype: motor_grader` to GPT-4o. The prompt heading is `GRADER`, not `MOTOR GRADER`. The template body says "Motor Grader" internally, but the heading breaks the exact-match pattern established in Phase 19 (WASHING, PRIVATE, RECREATIONAL, BULLDOZER/CRAWLER TRACTOR all exact-match). No test asserts this routing path. GPT-4o will likely select correctly given context, but there is no CI protection and the pattern is inconsistent.

**Suggested fix:** Rename prompt heading `GRADER` → `MOTOR GRADER`. Add test: `expect(s).toContain('MOTOR GRADER')`.

**Gap 2 — pig / tag routing to DOG / PIG / TAG (DESCR-02)**

`trailer.pig` and `trailer.tag` route to the `DOG / PIG / TAG` combined heading via GPT-4o inference. The `dog` key is tested; `pig` and `tag` are not. The combined heading explicitly names all three — correct routing is highly reliable — but CI has no regression protection for these two paths.

**Suggested fix:** Add test assertions for `('trailer', 'pig')` and `('trailer', 'tag')` each asserting `toContain('DOG / PIG / TAG')`.
