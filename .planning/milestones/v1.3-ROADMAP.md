## Milestone Goal

Add Marine asset type, expand truck/trailer/earthmoving/general goods subtypes, fix description quality across all types, and resolve two deferred pre-fill bugs.

## Key Accomplishments

1. **Marine asset type** ‚Äî 25-field schema (HIN, LOA, Beam, Draft, Motor Type, Engine Hours, etc.), 3 subtypes (Boat, Yacht, Jet Ski), Anchor icon in tile grid, full AI extraction pipeline with 15 aiExtractable fields, MARINE + JET SKI description templates
2. **Subtype expansions** ‚Äî Truck updated to 15 subtypes (9 new body types, Other added, Rigid Truck/Crane Truck removed); Trailer updated to 11; Earthmoving updated to 12 (Bulldozer renamed from Dozer, Crawler Tractor + Other added); General Goods gets 5 categorical subtypes replacing the bare "general" catch-all
3. **Description footer enforcement** ‚Äî `normalizeFooter` applied post-generation on all asset types; strips any wrong-variant footer and reappends correct "Sold As Is, Untested & Unregistered." ‚Äî no exceptions
4. **13 new description templates** ‚Äî ALL_CAPS named templates for all 15 truck subtypes and all 12 earthmoving subtypes; specialised buyer fields for VACUUM TRUCK, CONCRETE PUMP, CONCRETE AGITATOR, EWP; DOZER renamed to BULLDOZER throughout
5. **Multi-line notes fix** ‚Äî `extractFreeformNotes` reworked to return all lines after the `Notes:` marker, not just the first line ‚Äî multi-line textarea display correct on asset return
6. **sendBeacon unmount flush** ‚Äî iOS back-button data loss fixed; `navigator.sendBeacon` to new authenticated `POST /api/inspection-notes` route replaces unreliable Server Action cleanup; guarantees persistence within 500ms of navigation
