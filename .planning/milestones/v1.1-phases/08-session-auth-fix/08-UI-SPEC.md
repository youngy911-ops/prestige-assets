## Interaction States

Only the login form is touched in this phase (post-login redirect destination). Existing states remain unchanged:

| State | Behaviour |
|-------|-----------|
| Login idle | Form visible, Sign In button enabled |
| Login loading | Loader2 spinner replaces button label, button disabled |
| Login error | Red error paragraph rendered above button (`role="alert"`) |
| Login success | Server redirects to `/` (asset list) — changed from `/assets/new` |
| Authenticated at /login | Middleware redirects to `/` before page renders — new behaviour |
| Unauthenticated at / | AppLayout + middleware redirect to `/login` — unchanged |
| Session expired | Silent redirect to `/login` on next navigation — unchanged |
