### E2E Flows

**Flow 1: Staff saves inspection notes**

1. `photos/page.tsx` (or `extract/page.tsx`, `review/page.tsx`) fetches `inspection_notes` from DB
2. Passes as `initialNotes` to `InspectionNotesSection`
3. Component seeds `structuredValuesRef` via `parseStructuredFields(initialNotes)` and `notesRef` via `extractFreeformNotes(initialNotes)`
4. Fields render `defaultValue` from refs
5. Changes debounce-save via `saveInspectionNotes` server action
6. Unmount flush calls `persistNotes()` synchronously before navigation completes

**Status: COMPLETE**

**Flow 2: Staff returns — finds saved values**

1. Staff navigates back to any of 3 host pages
2. Server re-fetches fresh `inspection_notes` from DB
3. All 3 pages pass value as `initialNotes` (photos line 117, extract line 60, review line 55)
4. `InspectionNotesSection` mounts fresh, refs seeded, `defaultValue` shows saved values

**Status: COMPLETE**

### Auth Protection

All 3 pages hosting `InspectionNotesSection` perform auth checks with `supabase.auth.getUser()` and redirect to `/login` before any asset data is loaded.
